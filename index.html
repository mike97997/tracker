Connect to SharePoint — do it yourself (~20 min)

Sign in with your work Microsoft account. Part C needs admin rights — as the company owner you almost certainly have them.

A Make the list ~5 min, in SharePoint

Open your site → New → List → Blank list, name it exactly Projects, and add these columns:

  • Area — Single line of text
  • Status — Choice (Active, Next up, On hold, Idea, Done)
  • NextStep — Single line of text
  • Target — Date · Priority — Choice (hi, md, lo) · Focus — Yes/No · UpdatedOn — Date
  • Memory — Multiple lines of text (holds your “where I left off”, notes, decisions — keep it)

Make a second list named BrainDump with one column: Category — Choice (note, idea, task, follow, ques, urgent). The built-in “Title” holds the project name / note text.

B Put the page online ~5 min, free

  • Free account at github.com; rename the file to index.html.
  • + → New repository → name tracker → Public → Create.
  • Add file → Upload files → drop in index.html → Commit.
  • Settings → Pages → Deploy from a branch, main/(root) → Save. Copy your address, e.g. https://YOURNAME.github.io/tracker/.

C Register the app & copy two IDs ~7 min

  • entra.microsoft.com → search App registrations → + New registration.
  • Name it; Redirect URI → Single-page application (SPA) → paste your address from B.
  • Click Register. The Overview page shows Application (client) ID = App ID and Directory (tenant) ID = Directory ID. Copy both.
  • API permissions → Microsoft Graph → Delegated → add Sites.ReadWrite.All + User.Read → Grant admin consent.

D Connect

Open your live page → Data → paste App ID, Directory ID, and your site link → Save & Connect. 🎉
